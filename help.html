이 프로그램은 무엇인가요?

설교 스튜디오는 AI 부서들이 협업해 설교문 초안을 만들어 주는 프로그램입니다. 큰 회사처럼 각 부서가 자기 분야를 조사하고, 편집국이 모든 보고서를 모아 한 편의 설교문으로 완성합니다.

💡 주제 입력
↓
🔤 원어연구부 · 🗞️ 사회이슈부 · 📜 역사자료부 · 🌍 해외자료부
⛪ 국내설교부 · 🔬 예화부(과학) · 📚 예화부(고전) · 📱 예화부(트렌드)
↓
✍️ 편집국이 통합 → 완성된 설교문 초안

모든 데이터는 이 컴퓨터 안에서만 처리·저장됩니다 (클라우드 API를 선택한 경우에만 해당 회사 서버로 전송).

1. 실행하기

🍎 맥에서

1sermon-studio 폴더의 「설교스튜디오 실행.command」를 더블클릭합니다.
2검은 터미널 창이 뜨고, 잠시 후 브라우저가 자동으로 열립니다. 터미널 창은 서버이므로 그대로 두세요. 터미널 창을 닫으면 프로그램이 종료됩니다.
처음 실행할 때 "확인되지 않은 개발자" 경고가 뜨면: 파일을 우클릭(또는 Control+클릭) → 열기 → 열기를 한 번만 해주면 됩니다.

🪟 윈도우에서

1sermon-studio 폴더 전체를 윈도우 컴퓨터에 복사합니다 (USB, 클라우드 등).
2「설교스튜디오 실행.bat」을 더블클릭합니다. Python이 없으면 설치 안내 페이지가 자동으로 열립니다 — 설치 시 "Add Python to PATH" 체크박스를 꼭 선택하세요.

2. 5분 튜토리얼 — 데모 모드로 흐름 익히기

AI를 연결하기 전에, 프로그램이 어떻게 돌아가는지 먼저 구경해 보세요.

1왼쪽 사이드바 아래 ⚙️ LLM 엔진 설정 → 백엔드 엔진에서 「데모 모드 (테스트용)」을 선택합니다.
2맨 위 설교 주제에 아무 주제나 입력합니다. 예: 광야에서 만나는 하나님의 은혜
3▶ 전체 부서 가동 버튼을 누릅니다.
4가운데 화면에서 부서 노드들이 차례로 파란색(작업 중) → 초록색(완료)으로 바뀌는 걸 지켜보세요. 끝나면 오른쪽에 설교문이 나타납니다. (데모 모드라 내용은 예시 문구입니다)
노드를 클릭하면 그 부서의 보고서를 볼 수 있고, 드래그하면 위치를 옮길 수 있습니다.

3. 진짜 AI(LLM) 연결하기

이제 진짜 AI를 연결해 봅시다. 쉬운 순서대로 네 가지 방법이 있습니다.

방법 A — LM Studio (권장 · 무료 · 이 맥에 이미 설치됨)

1응용 프로그램에서 LM Studio 앱을 엽니다.
2왼쪽 돋보기(🔍 Discover) 아이콘 → 검색창에 Qwen3 4B MLX 입력 → 목록에서 MLX 표시가 있는 4bit 모델의 Download를 누릅니다 (약 2~3GB, 수 분 소요).
3다운로드가 끝나면 왼쪽 초록 터미널 아이콘(Developer) 탭 → 상단에서 방금 받은 모델 선택 → Start Server 버튼을 누릅니다. "Status: Running" 이 보이면 성공입니다 (포트 1234).
4설교 스튜디오로 돌아와서: 백엔드 엔진 → 「LM Studio」 선택 → ⟳ 버튼 → "✓ 연결됨"이 뜨면 기본 모델에서 모델을 선택합니다. 끝!

방법 B — 모델 보관함에서 직접 다운로드 (safetensors + MLX)

1사이드바 📦 모델 보관함의 검색창에 qwen 4bit 또는 EXAONE 4bit mlx를 입력하고 🔎.
2결과에서 🟢 초록 점이 있는 모델(이 맥에서 실행 가능한 MLX 모델)의 받기를 누릅니다. 진행률이 아래에 표시되고, 완료되면 "보관 중인 모델"에 나타납니다. 파일은 sermon-studio/models/ 폴더에 저장됩니다.
3보관된 모델 옆의 ▶ MLX 버튼을 누르면 그 모델로 AI 서버가 켜지고 엔진이 자동 전환됩니다.
※ 처음 한 번은 "mlx-lm 미설치" 안내가 뜹니다 — 폴더의 「MLX 모델서버 시작.command」를 더블클릭하면 설치를 도와줍니다.
4모델 로딩에 수십 초 걸립니다. 잠시 후 ⟳를 눌러 "✓ 연결됨"을 확인하세요.

방법 C — Ollama

1ollama.com에서 앱을 설치합니다 (이 맥에는 이미 설치되어 있습니다).
2터미널을 열고: ollama pull qwen3:4b (모델 다운로드) → ollama serve (서버 시작. 앱으로 설치했다면 자동 실행됨)
3설교 스튜디오에서 백엔드 엔진 → 「Ollama」 → ⟳ → 모델 선택.

방법 D — 클라우드 API (품질 최고 · 유료/무료 티어)

컴퓨터 성능과 관계없이 최고 품질을 원할 때. API 키를 발급받아 붙여넣기만 하면 됩니다.

서비스키 발급 위치특징
Groqconsole.groq.com무료 티어, 매우 빠름
Anthropic (Claude)console.anthropic.com한국어 설교문 품질 우수
OpenAI (GPT)platform.openai.com널리 사용
Google Geminiaistudio.google.com무료 티어 있음
1백엔드 엔진에서 서비스 선택 → API 키 칸에 붙여넣기 → ⟳ → 모델 선택.
이 컴퓨터(M2 · 메모리 8GB)에 맞는 모델 크기
로컬 모델은 4bit 양자화 + 4B(40억 파라미터)급까지가 쾌적합니다. 7~8B급은 다른 앱을 모두 닫아야 돌아가고, 그보다 크면 실행되지 않거나 매우 느립니다.
추천: Qwen3 4B(한국어 좋음) · EXAONE 3.5 2.4B(LG, 한국어 특화) · gemma 3 4B

4. 설교문 만들기

1설교 주제를 입력합니다. 구체적일수록 좋습니다.
예: 고난 중에도 감사할 수 있는 이유
2(선택) 본문 말씀: 하박국 3:17-19 처럼 입력하면 원어연구부·역사자료부가 이 본문을 파고듭니다.
3(선택) 추가 지시사항: 대상·분량·형식을 알려주세요.
예: 주일 대예배, 30분, 3대지 강해설교, 장년 성도 중심
4실행 옵션을 정합니다:
  • 순차 실행 (기본) — 부서가 차례로 일합니다. 로컬 모델에 적합.
  • 병렬 실행 — 전 부서 동시 가동. 클라우드 API에서 빠릅니다.
  • 🔍 웹 검색 연동 — 켜면 각 부서가 실제 인터넷 검색 결과를 참고합니다.
  • 상위 결과 본문까지 수집 — 검색된 페이지의 본문까지 읽어 더 정확해집니다 (조금 느려짐).
5▶ 전체 부서 가동! 노드 색으로 진행 상황을 볼 수 있습니다: 회색=대기, 파랑=작업 중, 초록=완료, 빨강=오류
6완료되면 오른쪽 📜 최종 설교문 탭에 초안이 나타납니다. 부서 노드를 클릭하면 📋 부서 보고서 탭에서 근거 자료를 볼 수 있습니다.
지난 작업은 사이드바 📁 작업 기록에 자동 저장됩니다 (최근 30건). 클릭하면 다시 불러옵니다.

5. 부서 다루기

  • 부서 편집: 노드 오른쪽 위 ✎를 누르면 이름·역할 프롬프트·전용 모델·웹 검색 여부를 바꿀 수 있습니다.
  • 역할 프롬프트 안의 {topic} {scripture} {extra}는 입력값으로 자동 치환되는 자리표시자입니다. 지우지 마세요.
  • 전용 모델: 부서마다 다른 AI 모델을 쓸 수 있습니다. 예: 편집국만 큰 모델. (Ollama·LM Studio에서 잘 동작)
  • 부서 끄기: 편집 창에서 "이 부서 활성화" 체크를 해제하면 그 부서는 쉬게 됩니다.
  • 부서 추가: 상단 ＋ 부서 추가 — 예: "성경원어부", "찬양선곡부", "청년사역부" 등 자유롭게.
  • 배치: 노드를 드래그해 정리하고, 엉키면 배치 초기화.

8. 문제 해결 (FAQ)

"연결 안 됨"이 떠요
  1. 엔진(LM Studio 서버, Ollama 등)이 실제로 켜져 있는지 확인하세요.
  2. LM Studio는 Developer 탭에 Status: Running이 보여야 합니다.
  3. 켠 다음 ⟳ 버튼을 다시 누르세요.
부서가 빨간색(오류)이 돼요

노드를 클릭하면 오류 내용이 보입니다. 대부분 ① 엔진이 꺼짐 ② 모델 미선택 ③ API 키 오류입니다. 한 부서가 실패해도 나머지는 계속 진행됩니다.

너무 느려요
  • 로컬 모델은 부서당 수 분 걸릴 수 있습니다. 8개 부서 전체는 수십 분도 정상입니다.
  • 더 작은 모델(2~4B)을 쓰거나, 안 쓰는 부서를 끄거나, 클라우드 API(Groq 무료)를 쓰세요.
  • "상위 결과 본문까지 수집"을 끄면 검색 시간이 줄어듭니다.
화면이 이상하거나 새 기능이 안 보여요

브라우저에서 Cmd+Shift+R (윈도우: Ctrl+Shift+R)로 강력 새로고침하세요.

브라우저를 실수로 닫았어요

서버(터미널 창)가 살아 있다면 브라우저 주소창에 localhost:8787을 입력하면 됩니다. 터미널 창까지 닫았다면 실행 파일을 다시 더블클릭하세요.

모델을 지우고 싶어요
  • 모델 보관함에서 받은 것: sermon-studio/models/ 안의 해당 폴더 삭제
  • LM Studio에서 받은 것: LM Studio 앱의 My Models에서 삭제
  • Ollama: 터미널에서 ollama rm 모델명
설정·기록은 어디에 저장되나요?

부서 구성·설정·작업 기록은 브라우저 안(localStorage)에, "서버에 저장"한 설교문은 output/ 폴더에, 다운로드한 모델은 models/ 폴더에 저장됩니다. 모두 이 컴퓨터를 벗어나지 않습니다.

⚠ 꼭 기억하세요
AI가 만든 설교문은 초안입니다. AI는 그럴듯한 가짜 인용·통계·일화를 만들 수 있으므로, 연도·인명·인용문·통계는 반드시 직접 확인한 뒤 강단에서 사용하세요. 보고서에 표시된 출처 URL과 "[확인 필요]" 표시를 확인하는 습관을 들이시면 좋습니다. 이 도구는 목회자의 묵상과 신학적 분별을 대체하지 않습니다.

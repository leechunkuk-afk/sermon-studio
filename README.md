# ⛪ 설교 스튜디오 (Sermon Studio)

각 분야에 특화된 LLM "부서"들이 순차적으로 자료를 조사하고, 편집국이 이를 통합해
설교문 초안을 작성하는 로컬 웹 앱입니다. Open WebUI의 사이드바 + ComfyUI의
노드 그래프를 섞은 인터페이스입니다.

## 실행 방법

**맥**: `설교스튜디오 실행.command` 더블클릭 (처음엔 우클릭 → "열기").
**윈도우**: 폴더를 복사한 뒤 `설교스튜디오 실행.bat` 더블클릭
(Python 3 필요 — 없으면 안내 페이지가 자동으로 열립니다. 설치 시 "Add Python to PATH" 체크).

두 런처 모두 서버를 켜고 브라우저를 자동으로 엽니다. 이미 켜져 있으면 브라우저만 엽니다.
터미널에서 직접 실행하려면:

```bash
cd ~/클로드/sermon-studio
python3 server.py          # http://localhost:8787 에서 열림
```

**초보자용 사용설명서**: 앱 상단의 「❓ 사용설명서」 버튼 (또는 http://localhost:8787/help.html) —
실행법, LLM 다운로드·연결, 설교문 작성, 문제 해결까지 단계별 안내.

**화면 테마**: 캔버스 상단에서 🌙 다크 / ☀️ 라이트 / 📜 양피지 / 🌊 딥블루 선택 (자동 저장).

## 부서 구성 (기본)

| 부서 | 역할 |
|---|---|
| 🔤 원어연구부 | 본문 히브리어·헬라어 원어 분석, 번역이 놓친 뉘앙스 |
| 🗞️ 사회이슈부 | 주제와 연결되는 현대 사회 이슈 조사 |
| 📜 역사자료부 | 교회사·본문 배경·세계사 자료 조사 |
| 🌍 해외자료부 | 해외 설교자·신학자 관점, 해외 주석 |
| ⛪ 국내설교부 | 한국 교회 설교 경향, 구조·표현 제안 |
| 🔬 예화부 (학술·과학) | 과학 현상, 심리·사회학 연구 예화 |
| 📚 예화부 (인문·고전) | 문학·철학·전래동화·전승·고전 예화 |
| 📱 예화부 (SNS·트렌드) | SNS·미디어 문화, 세대 공감 예화 |
| ✍️ 편집국 (종합) | 모든 보고서를 통합해 설교문 초안 작성 |

- 노드의 ✎ 버튼으로 각 부서의 **역할 프롬프트, 전용 모델, 활성화 여부**를 수정할 수 있습니다.
- "＋ 부서 추가"로 새 부서를 만들 수 있습니다 (예: 성경원어부, 찬양선곡부).
- 노드는 드래그로 자유롭게 배치할 수 있습니다.

## LLM 백엔드 엔진

**safetensors 모델에 대해**: safetensors는 모델 가중치 "파일 형식"이라서 직접 실행되지 않고,
아래 엔진 중 하나가 읽어서 서빙합니다. Apple Silicon 맥에서는 **LM Studio** 또는 **MLX**가
safetensors를 가장 잘 돌립니다.

### 로컬 엔진
| 엔진 | 모델 형식 | 사용법 |
|---|---|---|
| **LM Studio** ⭐ | safetensors(MLX)·GGUF | 앱 실행 → 모델 다운로드 → 개발자 탭에서 Start Server → ⟳. 이 맥에 이미 설치됨 |
| **MLX (mlx-lm)** | safetensors | 폴더의 `MLX 모델서버 시작.command` 더블클릭 → HuggingFace 모델 ID 입력 |
| **Ollama** | GGUF | `ollama serve` 실행 후 ⟳ |
| **llama.cpp** | GGUF | `llama-server -m 모델.gguf` |
| **vLLM** | safetensors | 리눅스/GPU 서버 권장 |
| **Text-Gen-WebUI / KoboldCpp** | 다양 | 각 서버 실행 후 선택 |

### 클라우드 API
Anthropic (Claude) · OpenAI (GPT) · Google Gemini · Groq (무료 티어, 매우 빠름) ·
OpenRouter (키 하나로 수백 개 모델) — API 키만 입력하면 모델 목록이 자동으로 뜹니다.

### 📦 모델 보관함 (모델 다운로드·보관·실행)

사이드바의 "모델 보관함"에서:
1. **검색** — HuggingFace에서 모델을 검색합니다 (🟢 표시 = 이 맥에서 돌아가는 MLX 모델)
2. **받기** — `sermon-studio/models/` 폴더에 다운로드합니다 (진행률 표시)
3. **▶ MLX** — 보관된 safetensors 모델로 MLX 서버를 바로 켜고 엔진을 자동 전환합니다
   (mlx-lm 미설치 시 안내가 뜹니다 — `MLX 모델서버 시작.command`로 설치)

모델 삭제는 `models/` 안의 해당 폴더를 지우면 됩니다.

### 부서별 전용 모델

각 부서 노드의 ✎ 편집에서 "전용 모델"을 **연결된 엔진의 모델 목록에서 선택**할 수 있습니다.
예: 원어연구부·편집국은 큰 모델, 예화부는 작은 모델. 비우면 기본 모델을 씁니다.
- 부서마다 다른 모델은 **Ollama·LM Studio**에서 잘 동작합니다 (요청한 모델을 자동 로드).
- **MLX(mlx-lm) 서버는 한 번에 한 모델만** 서빙하므로 부서별 모델 분리가 안 됩니다.
- 8GB RAM에서는 큰 모델 2개를 번갈아 로드하면 매우 느려집니다 — 부서별 분리는
  작은 모델들끼리 하거나 클라우드 API에서 쓰는 것이 현실적입니다.

### 이 맥(M2 · 8GB RAM)에 맞는 모델 크기
4bit 양자화 기준 **4B급까지가 쾌적**하고 7~8B급은 다른 앱을 닫아야 돌아갑니다.
추천: `mlx-community/Qwen3-4B-4bit`(한국어 좋음), `mlx-community/EXAONE-3.5-2.4B-Instruct-4bit`(LG 한국어 특화),
LM Studio에서는 검색창에 "Qwen3 4B MLX" 등으로 검색.

## 🔍 웹 검색 연동

- 사이드바의 "웹 검색 연동"을 켜면 각 부서가 LLM 호출 전에 **DuckDuckGo 검색**
  (API 키 불필요)을 실행하고, 검색 결과(제목·URL·요약)를 프롬프트에 넣어 조사합니다.
- "상위 결과 본문까지 수집"을 켜면 상위 2개 페이지의 본문 텍스트(최대 2,500자)까지
  가져와 훨씬 구체적인 조사가 됩니다 (부서당 수 초 느려짐).
- 부서별 검색어는 ✎ 편집 모달의 "검색어 템플릿"에서 수정합니다
  (`{topic}`, `{scripture}` 치환). 부서별로 검색을 끌 수도 있습니다.
- 프롬프트에 "활용한 웹 자료는 출처 URL을 표기하라"는 규칙이 포함되어 있습니다.

## 실행 방식

- **순차 실행** (기본): 부서가 차례로 일합니다. 로컬 Ollama에 적합
  (Ollama는 어차피 요청을 한 번에 하나씩 처리하므로 병렬의 이득이 없음).
- **병렬 실행**: 모든 조사 부서가 동시에 가동됩니다. Claude/OpenAI API 사용 시
  전체 시간이 크게 단축됩니다. 단, 웹 검색도 동시에 8건 나가므로 DuckDuckGo가
  일부 요청을 제한할 수 있습니다 — 검색 적중률은 순차 실행이 더 좋습니다.

## 내보내기

- **.md** — 마크다운 파일 다운로드
- **.docx** — Word 문서로 변환해 다운로드 (한글·MS워드에서 열림)
- **.pptx** — 예배 프로젝션용 PowerPoint 슬라이드로 변환 (다크 배경 + 골드 제목).
  표지(제목+본문 구절) 다음 소제목(`##`)마다 새 슬라이드가 만들어지고,
  긴 문단은 문장 단위 불릿으로, 한 슬라이드 7줄이 넘으면 "(계속)" 슬라이드로
  자동 분할됩니다. Keynote·PowerPoint에서 열어 디자인을 다듬어 쓰세요.
- **서버에 저장** — `sermon-studio/output/` 폴더에 저장

## 알아둘 점 (정직한 한계)

- 웹 검색을 꺼두면 각 부서는 **모델이 학습한 지식만으로 "조사"합니다.**
  프롬프트에 "불확실한 사실은 [확인 필요]로 표시하고, 출처를 지어내지 말라"는
  규칙이 들어 있지만, **연도·인용·통계는 반드시 직접 검증 후 사용하세요.**
  LLM은 그럴듯한 거짓 인용을 만들 수 있습니다. 웹 검색을 켜도 검색 결과의
  신뢰성(개인 블로그 등)은 별개이므로 출처 URL을 확인하세요.
- 로컬 소형 모델(8B 이하)은 한국어 설교문 품질에 한계가 있습니다. 초안 생성 →
  직접 다듬기 흐름으로 쓰는 것을 권장합니다.
- 결과물은 초안입니다. 신학적 검토와 목회자의 묵상을 대체하지 않습니다.

## 데이터 저장 위치

- 설정·부서 구성·작업 기록(최근 30건): 브라우저 localStorage
- "서버에 저장" 버튼: `sermon-studio/output/` 폴더에 `.md` 파일로 저장
